SQL Server 2012 管理及开发新功能体验;Note = eshang

1.T-SQL
支持代码片段与调试设置断点，整合入VS

2.新的还原选项
突然断点的恢复，由于不可抗拒力量引起，恢复方式：
刷LOG
；
数据迁移，从老版本SQL Server到新版本的在线迁移用以诱发数据丢失

3. 增强分区
支持最多15000个分区，之前版本是1000，更简洁的分区管理方式
SQL 2000的时候单表数据超过10万，性能就会明显下降
现在数据使用方式是使用现有数据来分析未来的发展方向，用些来划分辅助数据文件，放置在不同的分区，同样过多的数据分区也消耗性能，而这需要折中考虑
大量的I/O操作或者数据需要持久化处理的时候不要使用固态硬盘

4.列存储索引(APOLLO), 使用的时候需要有超大的内存支持128GB，讲师不了解与MongoDB的对比
数据在硬盘上还是行存储
优点是就算使用不好也不会带来性能降低，最多不提高性能，索引与分区使用不好会降低性能
聚集索引根据索引项来排列数据物理位置，非聚集索引只是简单的索引
注意：建立列存储索引之后该表即被封存，不能插入语更新。如果一定要更新有三种方法：
  a.禁用，更新，重建
  b.使用分区，交换分区
  c.Union all 操作
列存储可以用来提高数据仓储的查询速度

5.数据压缩：SQL 2008 能达到2-5倍比例
            SQL 2012 使用新的压缩算法，VertiPAQ, 可以提高至10几倍，也是列存储使用的压缩技术

知识驱动
语义分析
知识发现
开放和可扩展性
容易使用
；
功能具有改错，补全等

使用案例，例如市场调查报告，不同的姓名的大小写，城市名称的缩写不同，调表内容的空余项等得统一化与规范化；将调查报告自动合并为以家庭为单位的数据集合
